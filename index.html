  • Reap the rewards of a pattern-driven design process

The best laid plans...

Pattern libraries or style guides are a powerful tool to support delivery of consistently great digital experiences.

But they need to evolve with your products. If the patterns live outside your daily workflow updates will become a chore and make maintenance expensive. They will get neglected. And once out of sync with production – like the printed versions of yore – your style guide will become an irrelevance.

Streamline your pattern library workflow

Enter HTMLDoc – a build script for documented pattern libraries.

HTMLDoc allows you to put pattern library source code where it won’t get missed – inside your production workflow.

Get your docs in a row

  • Make demos of new UI patterns in your production HTML templates

  • Write pattern meta and annotations in HTML comment tags

  • Run HTMLDoc from the command line or in your CI pipeline*

  • Congratulations, you have a living, breathing style guide that people want to use

* (comments are safely stripped for production)
